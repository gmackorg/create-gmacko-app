# Initial Proposal

This is the first planning artifact for a new app or major feature.

## Workflow

1. Use `superpowers:brainstorming` to turn the raw idea into a concrete proposal and write the result into this file.
2. Run `/plan-ceo-review` to tighten the problem statement, user value, and scope.
3. Run `/plan-eng-review` to translate the approved scope into `docs/ai/IMPLEMENTATION_PLAN.md`.
4. Run `/design-consultation` to establish the design philosophy and generate `DESIGN.md`.

## Proposal Template

### Problem

What user problem are we solving, and why does it matter now?

### Users

Who is the primary user, and what are the highest-value moments for them?

### Core Experience

Describe the main workflow end to end in plain language.

### Scope

List the must-have capabilities for v1 and the explicit non-goals.

### Constraints

Capture technical, regulatory, platform, or business constraints that shape the plan.

### Success

State the product outcome that would make this launch a clear win.
